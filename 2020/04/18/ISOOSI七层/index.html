OSI七层模型

发表于 2020-04-18 更新于 2021-04-08

应用层

最复杂的一层，用户与网络的接口

常见服务：

  • FTP
  • SMTP
  • HTTP

表示层

用于处理在两个信道系统中交换信息的表示方式（语法和语义）

功能：

  • 数据格式变换
  • 数据的加密和解密
  • 数据的压缩和恢复

主要协议：HTML、ASCII

会话层

向表示层实体/用户进程提供建立连接，并在连接上有序的传输数据。用于建立同步（SYN）。

功能：

  • 建立、管理、终止会话
  • 建立校验点，通信失效时，从校验点恢复通信。（如传输大文件）

主要协议： 略

传输层

负责主机的两个进程间的通信，即端到端的通信。传输单位是报文段或用户数据包。

主要功能：

  • 可靠传输、不可靠传输
  • 差错控制：报文丢失、错误，报文顺序错误
  • 流量控制：端到端之间的传输流量，控制发送方的发送速率。
  • 复用：多个应用层同时使用下层运输层的服务，通过端口号进行区别。
  • 分用：通过不同的端口号进行识别，分别交付，不同的端口号之间互不干扰。

主要协议：TCP、UDP

网络层（IP层）

把分组从源端传送到目的端，为分组交换网上的不同主机提供通信服务。网际互连

网络层的传输单位：数据报

数据报过长时，会将数据报分割成小的分组（packet）。

功能：

  • 路由选择：选择路由器之间的传输路径
  • 流量控制：限制发送方的发送速度
  • 差错控制：保证传输层数据无错，出错分组丢弃。
  • 拥塞控制：宏观控制，控制全局。如果所有节点都来不及接收分组，将要丢失大量分组时，网络处于拥塞状态，需要采取一定措施来缓解。

数据链路层

把网络层传下来的数据报组装成帧（注：只有链路层会加头加尾）

链路层的传输单位是帧

功能：

  • 成帧（定义帧的开始和结束）
  • 差错控制：帧错、位错
  • 流量控制：控制发送方发送速率
  • 信道访问控制

常见协议：HDLC PPP SDLC STP

物理层

将比特流转换成电信号，在物理媒体实现比特流的透明传输

物理层传输单位是比特

功能：

  • 定义接口特性：引脚数目、插头
  • 定义传输模式：单工通信、半双工通信、全双工通信
  • 定义传输速率
  • 比特同步：发送方发送比特之后，接收方能够根据一定时序接收比特。
  • 比特编码：曼彻斯特、差分曼彻斯特编码
