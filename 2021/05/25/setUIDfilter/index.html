创建进程时setUID的过滤机制

发表于 2021-05-25

上次记录到了frameworks/base/core/jni/com_android_internal_os_Zygote.cpp文件中的SpecializeCommon函数，这个函数会对zygote新fork出来的进程进行一些特化，我关心的就是setresuid(uid, uid, uid)这个函数。不过之前也记录了，并不是只有app启动是通过zygote进行fork出来的，许多系统进程也是通过同样的方式fork的，所以如果想进行权限判断就需要对UID进行一定的规则过滤。

在setresuid之前，有一个SetUpSeccompFilter(uid, is_child_zygote);这个看起来很像是安卓自身进行规则过滤使用的。他的注释如下：

Must be called when the new process still has CAP_SYS_ADMIN, in this case, before changing uid from 0, which clears capabilities. The other alternative is to call prctl(PR_SET_NO_NEW_PRIVS, 1) afterward, but that breaks SELinux domain transition (see b/71859146). As the result, privileged syscalls used below still need to be accessible in app process.
